# 사용자 인증 기법
## 지식 기반 인증
#### 개요
- **사용자가 알고 있는 것에 의존하는 인증기법**
- ex) 패스워드 복구 질문 같은 것들
- 사용자 -> 신원정보 제공
- 이를 통해 사용자가 맞는지 확인
- 비밀번호 크기와 랜덤성에 따라 보안성이 다름
#### 장단점
- ##### 장점
	- 다양한 분야에 사용 가능
	- 검증 확실
	- 관리비용 저렴
- ##### 단점
	- 패스워드 망각 가능
	- 공격자가 추측 가능
	- 사회 공학적 공격 취약
### 패스워드
#### 개요
- 로그인에 사용
- 고정 패스워드
- 일회용 패스워드 (OTP)
#### 고정된 패스워드
- 접속 시 반복해서 사용되는 패스워드
- ##### 첫번째 방법
	- 표나 파일에 아이디, 패스워드를 저장
- ##### 두번째 방법
	- 평문 대신 패스워드 해시를 저장
- ##### 세번째 방법
	- 평문 + salt 값을 해시로 만들어 저장
#### 일회용 패스워드 (OTP : One Time Password)
- 한번 사용하면 재사용 불가
- 보안이 중요한 곳에 사용
- ##### 첫번째 방법
	- 사용자와 시스템이 패스워드를 1번만 쓰기로 합의
- ##### 두번째 방법
	- 패스워드를 순차적으로 업데이트
	- $P_i$로 $P_{i+1}$ 생성
- ##### 세번째 방법
	- 순차적 업데이트 + 해시 함수
	- lamport 해시함수
		- 패스워드를 이용하여 해시함수를 n번 계산함 ($H_n$)
		- 사용자가 패스워드를 이용하여 해시함수를 n-1번 계산한 코드를 보냄 ($H_{n-1}$)
		- 사용자에게 받은 코드를 해시함수를 한번 돌려 서버의 저장값과 비교 ($H_n = H(H_{n-1})$)
		- 서버에 사용자가 보낸 해시 코드 저장($H_{n-1}$)
		- 위 내용 반복
#### 패스워드 인증의 문제점과 보안 정책
- ##### 문제점
	- 개인정보로 추측하기 쉬움
	- 캐리킹 툴로 크랙 가능
	- 무작위는 사용자가 기억하기 어려움
- ##### 보안 정책
	- 최소 8자리 문자와 4가지 문자 사용
	- 동일 패스워드를 재사용하면 안됨
	- 로그인 시도 횟수를 제한
	- 휴면 계정 사용 불가능으로 변경
#### 안전성
- 패스워드 길이가 짧을수록(S), 사용 기간이 짧을수록(L), 빈도가 낮을수록 추측이 어려움(R)
$$P=\frac {L\times R}{S}$$
### 시도-응답 개인 식별 프로토콜
#### 개요
- 패스워드를 도청할 경우 안전하지 않음
- 대칭형 암호와 공개키 암호에 기반
- 비밀 정보를 자신만 알고 있는 사실을 보여주는 프로토콜
#### 일방향 개인 식별 프로토콜
- 서버나 클라이언트 중 한 대상을 식별하는 프로토콜
- 개인의 정체성을 노출하지 않고 인증
- ##### 사용하는 기술
	- Zero-Knowledge Proof
		- 비밀을 알고 있는 것을 증명하고, 비밀은 말하지 않음
	- 해시 기반 인증
		- 정보를 해시함수에 통과시킨 결과만 넘김
	- 익명 인증 프로토콜
		- 임시 토큰을 전달

#### 상호 개인 식별 프로토콜
- 클라이언트가 서버의 신원을 확인하는 경우

### 영지식 개인 식별 프로토콜
#### 개요
- 클라이언트가 서버에게 비밀 정보를 알고 있다는 사실을 입증하는 프로토콜
- 비밀 정보를 서버에 제공하지 않고 신분을 증명
#### 알고리즘
- 클라이언트 - 증명자
- 서버 - 검증자
1. 적당한 집합을 정함
2. 증명자가 난수에 대한 증거를 검증자에게 제시
3. 검증자는 여러 질문을 증명자에게 제시
4. 증명자는 모든 질문에 대답함으로써 정당한 클라이언트임을 확인

### i-PIN (Internet Personal Identification Number)
#### 개요
- 인터넷상 주민번호 대신 아이디 패스워드로 본인 확인
- 주민번호 없이 회원가입 가능
#### 이용효과
- 주민등록번호 유출예방
- 본인확인 강화

## 소유 기반 인증
#### 개요
- 열쇠, 면허증, 신분증 등 여러 매체를 통해 사용자 인증
- 쉽게 도용이 가능해 다른 방식들과 함께 사용
#### 장단점
- ##### 장점
	- 일반적임
	- 입증된 기술
	- 경제적
- ##### 단점
	- 없을 시 인증의 어려움
	- 복제 가능
	- 자산 관리 기능 필요
#### 메모리 카드(token)
- 저장 가능하지만 프로세스 데이터가 아님
- 마그네틱선이 있는 카드
#### 스마트 카드
- 정보를 저장 가능단 점에서 메모리 카드보다 발전
- 특정 업무 처리 가능
- 장비 보호 기능 필요
#### 일회용 패스워드(OTP)
- 일정 시간마다 비밀번호 변경
- 사용자 편리성 및 안전성 확보
- ##### 인증 방식
	- **질의응답**
		- 서버 -> 사용자 에게 nonce값 전송
		- 임의값을 토큰장치에 입력
		- 토큰장치가 패스워드를 알려줌
		- 서버로 전송
		- 일치하면 사용자 인증
	- **시간과 이벤트 동기화**
		- ###### 시간 동기화
			- 생성값과 ID를 컴퓨터 입력
			-  시간 기반 토큰을 통해 패스워드 생성
		- ###### 이벤트 동기화
			- 토큰장치 버튼을 눌러 생성
			- 사용자에게 보여진 값을 입력
	- **S/Key 방식**
		- 클라이언트가 정한 비밀키를 서버로 전송
		- 받은 비밀키를 해시 체인 방식으로 n번 반복
		- 생성된 n개의 OTP 저장

## 개체 특성 기반 인증
#### 개요
- 지문, 홍채, 장문, 성문 등 이요
- 생체정보를 DB에 저장
- 검증과 신분확인에 사용
#### 장단점
- ##### 장점
	- 확인이 쉬움
	- 도난 불가
	- 위조가 어려움
- ##### 단점
	- 잘못 판단할 가능성 존재
	- 관리의 어려움
	- 임계치 설정이 어려움
#### 생체 인증
- ##### 개요
	- 생체적, 행동적 특성으로 신원을 인증
	- 훔치거나 공유 불가
- ##### 평가 항목
	- 보편성 - 다 가지고 있는지
	- 유일성 - 같은 특징인 사람이 없는지
	- 지속성 - 변화가 없는지
	- 획득성 - 측정이 가능한지
	- 성능 - 높은 정확성을 가지는지
	- 수용성 - 거부감이 없는지
	- 반기만성 - 부정사용에 안전한지
- ##### 정확도
	- 지나치게 민감하면 사용자를 잘못 판단할 수 있음
	- 민감도가 떨어지면 비정상 사용자를 정상 사용자로 판단할 수 있음
	- -> 민감도와 오류율이 같아지도록 유지
