# 대칭키 암호

대칭키 암호(Symmetric Key Encryption)는 암호화와 복호화에 동일한 키를 사용하는 암호화 방식입니다. 이는 가장 오래되고 널리 사용되는 암호화 기법 중 하나로, 데이터의 기밀성을 보장하는 데 효과적입니다. 대칭키 암호는 빠른 속도와 효율성으로 인해 다양한 응용 분야에서 널리 활용되고 있습니다.

## 현대 대칭키 암호
#### 현대 블록 암호의 구성요소
- 이동요소
- 교환요소
- 분할요소
- 조합요소
- ##### 전치장치 (P-box)
	- 고전 전치 암호를 병렬적으로 수행
	- ###### 축소 P-box
		- n비트를 입력받아 m비트를 출력 (n>m)
		- 다음 단계에 비트 수를 줄일 때 사용
	- ###### 확장 P-box
		- n비트를 입력받아 m비트를 출력 (n<m)
		- 다음 단계에 비트 수를 늘일 때 사용
	- ###### 역함수의 존재
		- 단순 P-box -> 존재
		- 축소, 확장 P-box -> 없음
	- ![[Pasted image 20250408152656.jpg]]
- ##### 치환장치 (S-box)
	- 입출력 개수가 다를 수 있음
	- 역함수가 존재 -> 이때는 입출력 개수가 같음
- ![[Pasted image 20250408152754.jpg]]
- XOR 연산

#### 합성 암호
>[!tips] 개요
>Shannon이 소개한 치환, 전치, 그 밖의 구성요소를 결합한 복합적인 암호

##### 확산과 혼돈
- 확산
	- 암호문과 평문 사이의 관계를 숨기는 것
- 혼돈
	- 암호문과 키의 관계를 숨기는 것
##### 라운드
- s-box, p-box, 기타 구성 요소의 결합을 반복함으로써 확산과 혼돈을 얻을 수 있음
- 이를 라운드라고 함
#### 합성 암호의 종류
- ##### Feistel 암호
	- 암호 강도 결정 요소
		- 평문 블록 길이 - **64비트 이상**
		- 키 길이 - **128비트 이상**
		- 라운드 수 - **16회 이상**
	- 특징
		- 입력 n비트를 두 블록으로 나눈다 $(L_0, R_0)$
		- 라운드 함수를 $F$, 라운드 키를 $K_i$라고 할 때 $i$번째 라운드는 다음과 같다
			- $L_i = R_{i-1}$
			- $R_i = L_{i-1}\oplus F_i(R_{i-1},K_i)$
	- ![[Pasted image 20250408152507.jpg]]
- ##### SPN 구조
	- Substitution-permutation network
	- 여러 함수를 중첩하면 개별 함수보다 안전하다
	- ###### 작동방법
		- 입력을 여러 소블록으로 나누고, 소블록을 s-box로 입력
		- s-box출력을 p-box로 전치
	- ![[Pasted image 20250408152552.jpg]]
#### 블록 암호 공격
- ##### 차분 공격 (Differential Cryptanalysis)
	- 두 평문 블록이 대응되는 두 암호문 블록으로 변환
	- 평문과 암호문 블록 사이의 비트 차이를 이용해 암호키를 찾는 방법
- ##### 선형 분석 (Linear Cryptanalysis)
	- 비선형 구조 -> 선형화하여 키를 찾는 방법
	- 근사적 선형 관계식을 찾아 이를 이용하여 chosen plaintext 공격 수행
- ##### 전수공격법 (Exhaustive key search)
	- 암호화할 때 일어날 수 있는 모든 경우를 조사
	- brute force attack
- ##### 통계적 분석 (Static analysis)
	- 통계학적 자료를 이용하여 해독
	- ex) 어떤 단어가 가장 많이 쓰였는지 분석
- ##### 수학적 분석
	- 통계적 방법 + 수학적 이론

#### 현대 스트림 암호
- ##### 기본 개념
	- 암, 복호화가 한 번에 *r비트*를 생성
		- 평문 스트림 - $P=p_n ... p_2p_1$
		- 암호문 비트 스트림 - $C = c_n...c_2c_1$
		- 키 비트 스트림 - $K = k_n ... k_2k_1$
		- $p_i, c_i, k_i$는 *r 비트워드*
			- r 비트워드 - 한번에 생성하는 비트 수
		- 암호화는 $c_i=E(k_i,p_i)$
		- 복호화는 $p_i=D(k_i,c_i)$
	- **키 스트림을 어떻게 생성하는지가 핵심**
- ##### 스트림 암호 설계 고려사항
	- 암호화의 연속은 긴 주기를 가져야 함
	- 키 스트림은 *진 난수 스트림*과 비슷해야 함
		- -> 각 비트가 나온 횟수가 비슷해야 함
	- 키가 충분히 길어야 함
- ![[Pasted image 20250408152900.jpg]]
#### 동기식 스트림 암호
- 키 스트림이 평문과 관계없이 생성되어 정보 유출 가능성이 낮음
- ##### One-Time Pad
	- 암호화시 랜덤 키 스트림 사용
	- 암호화 알고리즘과 복호화 알고리즘은 배타적 논리합 연산(XOR) 사용
- ##### 귀환 시프트 레지스터 (Feedback Shift Register)
	- OTP의 절충안
	- 하드웨어 구현이 용이
- ##### 선형 귀환 시프트 레지스트 (Linear Feedback Shift Register)
	- 많은 스트림 암호에서 사용
	- 선형성 때문에 공격에 취약
- ##### 비선형 귀환 시프트 레지스트 (NonLinear Feedback Shift Register)
	- LFSR보다 안전한 스트림 암호 설계 가능
